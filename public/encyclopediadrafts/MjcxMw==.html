Allegro Software Development Corporation

Allegro Software Development Corporation is an American software company specializing in secure, embedded Internet of Things (IoT) connectivity and device management solutions. Headquartered in Boxborough, Massachusetts, the company is known for its pioneering work in embedded web server technology and IoT security toolkits. Allegro’s software products are embedded in over 275 million devices globally across various sectors, including medical, industrial automation, home networking, and automotive systems.[1][2]

Products

Allegro Software offers a comprehensive suite of embedded Internet toolkits that are designed specifically for resource-sensitive environments with limited resources. Unlike repurposed PC applications or open-source tools, Allegro’s offerings are optimized for small memory footprints, efficient CPU usage, and compliance with modern security and web standards (HTTP/1.1, TLS, SSH).[2] These include embedded web servers, XML frameworks, secure cryptographic libraries, UPnP/DLNA stacks, and remote management solutions.[6][7][8]

IoT edge device security

ACE (Allegro Cryptography Engine): A FIPS 140-3 validated, high-performance cryptographic library optimized for embedded platforms. It provides capabilities to implement secure boot, TLS communication, secure data storage, and cryptographic operations with minimal resource usage.[4][7]

RomSTL: A lightweight, embedded TLS version 1.2/1.3 client/server specifically engineered for IoT environments. It supports encrypted data transport in security-sensitive applications.[7]

RomSShell: A Secure Shell (SSH) implementation that enables secure, encrypted remote command-line access. Often used with RomCLI for secure remote management.[7]

RomCert: A certificate management module that streamlines the handling of X.509 certificates in IoT devices, enabling secure authentication and encrypted communication.[7]

Java Cryptography Extension (JCE) from Allegro: An extension based on ACE, offering Java developers access to the same embedded-grade cryptographic capabilities in Java environments.[7]

Embedded internet toolkits

RomPager: A compact, high-performance embedded web server used widely in networked devices since 1996. It enables standard web browser access to device configuration and status interfaces without the need for custom applications.[6]

RomWebClient: An embedded HTTP client that allows IoT devices to communicate with web servers using HTTP over IPv4 and IPv6 networks. This toolkit supports object transmission and reception for seamless web integration.[6]

RomXML: A lightweight XML parsing and framing toolkit for embedded systems. It enables interoperability between heterogeneous systems via processor-independent XML encoding.[6]

RomXML Plus: An advanced version of RomXML tailored for building enterprise-grade IoT applications. It supports XML web services (SOAP, REST, JSON) for scenarios like industrial automation, sensor monitoring, and remote data collection.[6]

RomCLI: A command-line interface toolkit supporting Cisco IOS-style CLIs for embedded devices. It can operate independently or alongside RomPager for unified variable access and device management.[6]

RomMailer / RomPOP: Email agents designed to send alerts or receive configuration data. These modules enable email-based monitoring and remote updates in embedded applications.[3][6]

UPnP / DLNA toolkits

RomPlug Core Development Toolkits: A suite designed to enable UPnP and DLNA interoperability in connected consumer electronics and mobile devices. RomPlug supports device discovery, control, and media sharing in networked home environments.[8]

RomPlug DTCP-IP: A specialized version of RomPlug that integrates DTCP-IP (Digital Transmission Content Protection over IP), supporting the secure transfer of copyrighted digital content within home networks.[8]

RomPlug UPnP-DLNA Application Toolkits: These enable robust development of networked applications for smart TVs, media servers, and other connected devices. RomPlug ensures seamless integration with existing DLNA ecosystems.[8]

Solutions

Allegro’s secure IoT stack addresses the full lifecycle and security management of edge devices. Solutions include:

These security solutions are integrated into various embedded applications to ensure end-to-end device protection.

Algorithms 

Allegro Cryptographic Engine (ACE) has been validated under the Cryptographic Module Validation Program (CMVP) to meet FIPS 140-3 Level 1 standards. ACE is a software-based cryptographic module designed for embedded systems and consumer electronics, providing functions like encryption, hashing, digital signatures, and key management. A full description of the ACE module and latest NIST validations can be found by reviewing the ACE Security Policy found on the NIST website (FIPS 140-3 Validation #4944). Platforms

Allegro Software’s connectivity and security toolkits are platform-agnostic and have been ported to run on many enterprise and real-time operating systems.  The software toolkits are delivered as ANSI-C source code and are highly configurable to meet each customer’s unique application needs. The toolkits have successfully been deployed on 16-bit, 32-bit, and 64-bit silicon architectures across a broad range of operating system environments. A few include Microsoft Windows,  Linux, macOS, ThreadX, VxWorks, FreeBSD, NetBSD, OpenBSD, embedded Linux, OpenEmbedded, WinCE, Haiku, iPhone, Android, QNX, MontaVista, Tron variants, MicroC/OS-II, FreeRTOS, SafeRTOS, Freescale MQX, Nucleus, embOS, mbed, Green Hills INTEGRITY, and others.

History

Founded in February 1996, Allegro Software Development Corporation was established to bring the advantages of standard Internet applications to network device management.

1996 -1999: Early innovations 

1996: RomPager® launched, allowing embedded device vendors to use standard web browsers for device management without custom client software.

1997: Introduced RomPager® 2.0, RomMailer, and RomPager® Remote Host, adding HTTP 1.1 support and embedded email capabilities.

1998: Released RomPOP, enabling devices to receive emails for tasks like software updates and email printing.

1999: Version 3.0 brought enhancements to existing products and introduced RomWebClient™ for HTTP object retrieval and RomDNS for DNS queries. RomXML®, a compact XML parser for embedded devices, also debuted.

2000 - 2003: Expanding protocols and security

2000: Launched RomUPNP toolkits for UPnP 1.0 protocol and RomCLI™ for CLI development.

2001–2003: Introduced secure versions of core products: RomPager® Secure (SSL/TLS), RomWebClient™ Secure, and RomCLI™ Secure (SSH). The RomUPNP line was rebranded as RomPlug® with the addition of Media kits for UPnP AV.

2003: Partnered with Philips to bring RomPlug® to the Nexperia semiconductor line.

2004 - 2006: Media and OEM development tools

2004: Debuted Allegro Media Server for bridging iTunes with UPnP players and launched RomPlug® MediaControl.

2005: Continued releasing RomPlug® Device Toolkits for rapid OEM deployment.

2006: Microsoft selected Allegro to provide UPnP/DLNA and security technology for its Media Center Extender SDK, used in the Xbox 360.

2007 - 2010: Enhanced functionality and DLNA leadership

2007: Released RomPager® 4.6, updates to RomPlug®, the RomTime SNTP toolkit, and RomXOAP, enabling embedded SOAP integration.

2008: Added toolkits for DTCP-IP and WMDRM-ND link protection.

2009: RDMC 101 was chosen as the Reference Digital Media Controller for DLNA 3-box model certification.

2010: RomPlug® powered 50+ DLNA Certified products. Announced partnership with Express Logic, bundling RomPager® with the NETx TCP/IP stack.

2011 - 2013: Security integration and certification

2011: RDMS 202/202D became among the first DLNA Certified DMCs supporting Protected Streaming.

2012: Introduced RomCert™ for managing security certificates in embedded systems and integrated Suite B cryptography across the RomPager® product line.

2013: Initiated FIPS validation and earned FIPS 140-2 Level 2 certification for the Allegro Cryptography Engine (ACE™).

2014 - 2017: Advanced security and IoT readiness

2014: Released the Advanced Edition (AE) suite supporting IPv6 and FIPS-validated security; added support for DLNA's VidiPath initiative.

2015: Added pre-integrated encrypted data-at-rest features using ACE™.

2016: Allegro’s UPnP/DLNA stack powered award-winning wireless speaker systems.

2017: Earned additional FIPS validations for ACE™ in line with updated NIST guidelines.

2018 - present: Evolving with IoT and cloud integration

2018: Collaborated with the International Cryptography Module Conference to establish an IoT-focused cryptography track.

2019: Expanded ACE™ to support hardware-based entropy sources and earned further FIPS validations.

2020: Introduced support for TLS 1.3 and OpenSSL-compatible APIs, enabling integration with open-source investments. Announced compatibility with Amazon Web Services (AWS) IoT Gateway, leveraging FIPS-validated cryptography for secure, cloud-enabled communications.

2025: Allegro’s ACE cryptographic module was awarded FIPS 140-3 Interim Status.[9] 

Over the years, the company expanded its portfolio to include email agents (RomMailer, RomPOP), HTTP clients (RomWebClient), XML parsing (RomXML, RomXML Plus), UPnP/DLNA toolkits (RomPlug Core, RomPlug Application, RomPlug DTCP-IP), a highly configurable Command Line Interpreter (RomCLI), and secure access modules (RomSTL, RomSShell, RomCert, and the Allegro Cryptographic Engine).[3] 

Recognition 

Allegro has earned several industry recognitions, including multiple FIPS 140-2 validations from the U.S. National Institute of Standards and Technology (NIST) for its Allegro Cryptography Engine. The company is known for its strict adherence to secure development standards and for helping customers achieve DLNA and UPnP certification compliance in consumer electronics.[4]
